1. Dashboard
Go Back
Dashboard offers a visual access points to different pages within the project in a form of colorful boxes.

Each colorful box represents a different webpage the application, designed to help you navigate and manage your tasks efficiently. Whether you need to view list of players, edit player information, or view upcoming games & events, you can do so with ease directly from the dashboard. Simply click on any box to explore the corresponding page and take control of your workflow.

Here is the dashboard with boxes numbered from 0 to 10.
  • 0. Takes you to the player-list page where all senior players will be listed in table format.
  • 1. Takes you to the junior-list page where all junior players will be listed in table format.
  • 2. Takes you to the member-list page where all non-player members will be listed in table format.
  • 3. Takes you to the add-player page where you will be able to add a new senior player.
  • 4. Takes you to the add-junior page where you will be able to add a new junior player.
  • 5. Takes you to the add-member page where you will be able to add a new non-player member.
  • 6. Takes you to the timetable page where all events will be listed in a table format, with option to add, update and remove a event.
  • 7. Takes you to the address-list page where all address records will be listed in table format.
  • 8. Takes you to the guardian-list page where all guardian will be listed in table format.
  • 9. Takes you to the doctor-list page where all doctors will be listed in table format.
  • 10. Takes you to the coach-list page where all coaches will be listed in table format.
2. Header Bar
Go Back

The header bar provides easy access to key functionalities based on user's role within the system.

For administrators, it offers options to manage users, access to dashboard, and access administrative features.
For coaches, it offers slightly less options than admin, access to dashboard is revoked, but coach can still view, add, edit and remove players. They just don't have access to certain pages like list of addresses, guardians, coaches, etc.
Members, on the other hand, can view upcoming events and access their profile.

Admin Header



  • Dashboard: Takes you to the dashboard page.
  • Member Management: Dropdown with links to player,junior and member list pages
  • Timetable: Takes you to the timetable page.
  • Logout: Takes you to the logout page, where you will be logged out.

Coach Header



  • Profile: Takes you to your profile page.
  • Member Management: Dropdown with links to player,junior and member list pages
  • Timetable: Takes you to the timetable page.
  • Logout: Takes you to the logout page, where you will be logged out.

Member Header



  • Profile: Takes you to your profile page.
  • Timetable: Takes you to the timetable page.
  • Logout: Takes you to the logout page, where you will be logged out.

3. UI Elements
Go Back
UI elements play a crucial role in interacting with my application and performing various actions.

Regular buttons are used for common actions such as submitting forms, confirming choices, or triggering specific functions.
Additionally, checkboxes allow users to select single records from a list, such as selecting records as well as hiding specific columns in the settings modal.
Together with radio buttons, which allow users to select a single option from a list, these UI elements offer intuitive ways for users to interact with my application and accomplish their tasks efficiently.

Login/Register Buttons


  • Log In: Logs you in
  • Don't have an account? Click this link to register!: Takes you to the register page


Settings Button


  • Settings: Opens a settings modal with options to hide or show columns


List Checkboxes


  • Checkbox: Picks ID number of the selected record, later to be passed when form is submited


User Actions Buttons


  • Add Player: Takes you to the add-player page
  • Update Player: Takes you to the update-player page. Page to update/edit player's details
  • Update Player Skill: Takes you to the update-player-skill page. Page to update/edit player's skill details
  • Remove Player: Takes you to the remove-player page. Page to delete player record.


Squad/Coach Dropdowns


  • Dropdown Button: A dropdown button populated with SQL results of squad table. Used to assign player to squad


Radio Buttons


  • Radio Button: A radio button which determines whether there are one or two guardians to be added


File Buttons


  • File Button: A button which uploads selected image to the database, later to be used as a profile picture.

Some forms may be multi-stage, guiding users through a series of steps to complete a complex task or process. These multi-stage forms streamline the user experience, breaking down larger tasks into smaller, more manageable steps.

Whether it's viewing lists of records, analyzing metrics, or organising information, tables offer a clear and organized way to present data. The tables are designed with features such as hiding or showing columns to help users navigate large datasets easily.

Additionally, tables may include interactive elements such as clickable rows or action buttons for users to perform specific tasks directly from the table view. With user-friendly table designs and intuitive functionalities, users can efficiently access and manage data within the application.

6. Modals & Dialogs
Go Back

Modals and dialogs are interactive components used to display additional information, gather user input, or prompt for actions within the application.
These pop-up windows are hidden by default and only appear on webpage if button trigerring them is clicked, providing focused interactions without navigating away from the current page.
Modal windows typically contain forms, messages, or confirmations, allowing users to complete tasks or make decisions within a confined context.

Dialogs, on the other hand, serve as prompts for specific actions or alerts, such as confirming a deletion or notifying users of system updates.

With modals and dialogs, our application offers streamlined interactions and clear communication to enhance the user experience.
